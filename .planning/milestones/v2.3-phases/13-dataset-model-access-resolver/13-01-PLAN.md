<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stored `classification_override` value (seed/test fixture) -> `effectiveDatasetClassification`/`validateDatasetClassification` | Untrusted config data crossing into the classification derivation — must never silently permit an under-classification |
| `DatasetNode.application_ids` (string references) -> `effectiveDatasetClassification`'s Application lookup | Untrusted cross-reference; a non-existent id must fail closed, not resolve to a default |
| A stored `DatasetAccessGrant.level`/`ArchiveRole` value -> `isLevelInVocabulary`/`archiveRoleCovers`/aggregation functions | Untrusted data value that must be dispatched through the correct dataset-type vocabulary before any comparison, never compared bare |

| T-13-07 | Information Disclosure | `effectiveDatasetClassification`'s Application lookup | medium | mitigate | A dataset referencing a non-existent `application_id` throws a seed-integrity error rather than silently defaulting to a permissive (low) classification — Task 2 |
