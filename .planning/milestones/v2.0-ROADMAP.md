# Milestone v2.0: Authorization Hub (demo)

**Status:** ✅ SHIPPED 2026-05-22
**Phases:** 1–4
**Total Plans:** 16

## Overview

The demo is built in four phases. Phase 1 lays the data and engine foundation: a unified mock world-state,
a pure-computed ABAC engine, and the 8 operating roles — everything downstream views depend on.
Phase 2 builds the federation hub and entity consoles: discovery-without-disclosure, the typed interchange
contract, signed credentials, and holder-gated release. Phase 3 adds the audit and context/policy views:
the append-only log with point-in-time reconstruction, per-entity policy divergence, support-obligation
grants, and directional shielding. Phase 4 composes the four views into a coherent demo shell, validates
cross-view consistency, enforces the legibility gate, and registers the entry in the production build.

## Milestone Summary

**Key Decisions:**

- Demo island isolated from TanStack router (no routeTree changes) — prevents demo from coupling to app auth flow
- Pure-computed ABAC, no stored grants — audit log is system of record; O(1) materialized projection for current state
- Network class NOT ported from spikes; only pure functions in contract.ts (D2-02)
- ROGUE-ISSUER excluded from TRUSTED_ISSUERS to demonstrate untrusted-issuer rejection
- verify-before-trust enforced in ExchangeTranscriptPanel: verifyCredential before computeDetailResponse (D2-10)
- Deployment toggle in ContextView uses local useState — not dispatched to world-state (no stored grant created)
- UnitConsolePanel promoted from inside FederationHub to standalone Entity Console tab in Phase 4
