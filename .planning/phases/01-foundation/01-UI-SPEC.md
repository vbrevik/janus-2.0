# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** one isolated demo island (`frontend/src/demo/`, served by `demo.html`) containing a persistent `[DEMO / MOCK]` banner + operating-role switcher header, and a single **Decision Explorer** view (subject + resource + domain pickers → live ALLOW/DENY `DecisionTrace` → role-gated action panel). The contract reuses the proven spike visual vocabulary (`spikes/components/ui.tsx`) and the existing shadcn/Tailwind tokens verbatim — Phase 1 lifts, it does not reinvent. ## Spacing Scale

Declared values (must be multiples of 4). Maps directly to the Tailwind utility classes the spikes already use (`gap-4`, `p-4`, `space-y-4`, `px-2 py-0.5`, `px-3 py-1.5`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Pill/tag inline padding (`py-0.5`→2px vertical is the documented exception), icon-to-text gaps (`gap-1`) |
| sm | 8px | Compact element spacing (`gap-2`), button padding (`py-1.5`→6px is the documented exception) |
| md | 16px | Default element + card spacing (`p-4`, `gap-4`, `space-y-4`) — the dominant rhythm of the Decision Explorer |
| lg | 24px | Section padding between picker cards and the trace block |
| xl | 32px | Layout gaps / outer demo container padding |
| 2xl | 48px | Major section breaks between the header chrome and the view body |
| 3xl | 64px | Page-level top/bottom breathing room on wide viewports |

Exceptions (carried verbatim from the proven spike components — do not "tidy" these to the 8-pt grid, they are tuned for dense legibility per D-06):
- Pill/`MockTag` vertical padding `py-0.5` = 2px (`spikes/components/ui.tsx:21`).
- Action-button vertical padding `py-1.5` = 6px (`Spike004Sod.tsx`).
- `DecisionTrace` rule-row vertical gap `space-y-1.5` = 6px (`spikes/components/ui.tsx:95`).
- Rationale: these sub-grid values keep ~30 subjects/resources (D-06 rich seed) scannable without burying the decision trace.
- Guardrail: these three sub-grid values are the ONLY permitted off-grid spacing; any new spacing the executor adds MUST come from the 4/8/16/24/32/48/64 token set.

## Typography

Three roles + display. Sizes map to Tailwind `text-xs/sm/lg/xl`, matching the spike substrate exactly. Carrying weights: 400 body / 500 medium-emphasis (`font-medium`) / 600 semibold-heading (`font-semibold`). Exception: 700 bold (`font-bold`) on the single ALLOW/DENY verdict line only — no other element uses 700.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label (uppercase, tracked field/section labels) | 12px (`text-xs`) | 600 (`font-semibold`), `uppercase tracking-wide` | 1.4 |
| Body / control text | 14px (`text-sm`) | 400 regular; 500 (`font-medium`) for emphasis (rule names, actor names) | 1.5 |
| Heading (card titles, view section heads) | 18px (`text-lg`) | 600 (`font-semibold`) | 1.3 |
| Display (the ALLOW / DENY verdict) | 20px (`text-xl`) | 700 (`font-bold`) | 1.2 |

Notes:
- Body line-height 1.5, heading/display line-height ≤1.3 per default recommendation.
- The event/audit log uses `font-mono text-xs` (12px monospace) for timestamp + actor + action alignment (`Spike004Sod.tsx:167`).
- Do NOT add a 5th size or a 3rd carrying weight; the `[MOCK]`/`[DEMO]` tags and pills all live at `text-xs`.

## Color

The base palette is the existing shadcn slate theme (`src/index.css` CSS variables). **Semantic decision colors (green/red/amber/blue) are load-bearing signal, not decorative accent** — they encode ABAC outcomes and MUST be reserved for those meanings only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `0 0% 100%` (white) / page + view body surface; slate-50/100 for sub-surfaces | Background, view canvas, log/empty-state panels (`bg-slate-50`) |
| Secondary (30%) | `--card` white with `--border` `214.3 31.8% 91.4%` (slate-200); slate-700/400 for text | Cards, picker panels, the persistent header chrome, muted helper text (`text-slate-400/500`) |
| Accent (10%) | `--primary` `221.2 83.2% 53.3%` (blue) | Reserved list below |
| Destructive | `--destructive` `0 84.2% 60.2%` (red) | Deny overrides + destructive role actions only (see reserved list) |

Accent reserved for (explicit — never "all interactive elements"):
- The blue `--primary` accent is reserved for: the primary clearance/info `Pill` (`tone="blue"`) and focus rings (`--ring`). It is NOT used to paint every button.

Semantic decision palette (fixed meanings — the executor must not repurpose these hues):
- **Green** (`green-700` solid / `green-50` + `green-200` surface): ALLOW verdict, passing rule rows (`✓`), and the Approver **grant** action button.
- **Red** (`red-700` solid / `red-50` + `red-200` surface): DENY verdict, failing rule rows (`✗`), `⛔` deny-override lines, and the Security Officer **place/clear hold** action button.
- **Amber** (`amber-700` solid / `amber-100` pill): per-domain tier values and the Approver **revoke** action button (caution, reversible).
- **Slate** (`slate-700` solid / `slate-100` pill): neutral attributes (entity/unit, compartments) and the Manager **request** action button (no decision authority).
- **`[DEMO / MOCK]` banner + `[MOCK]` tag (MODEL-03):** amber/yellow caution treatment, visually distinct from the green/red decision palette so a simulated trust signal is never confused with a real ALLOW/DENY. High-contrast, non-dismissable.

## Copywriting Contract

All copy below is locked by CONTEXT/RESEARCH (it carries the legibility gate DEMO-03 and the SoD crux ROLE-02). The executor must use these strings verbatim.

| Element | Copy |
|---------|------|
| Primary CTA (Approver) | "Approve: grant {COMPARTMENT}" (green) and "Revoke: remove {COMPARTMENT}" (amber) |
| Secondary CTA (Security Officer) | "Place security hold" / "Clear security hold" (red, toggles on `securityHold` state) |
| Secondary CTA (Manager) | "Request {COMPARTMENT} (cannot grant)" (slate — logs only, zero mutation) |
| Empty state heading (event/audit log) | "No actions yet." |
| Empty state body (event/audit log) | "Auditor sees this log read-only; it is the system of record under pure-ABAC." |
| Empty state (role action panel — Admin/Sponsor/Subject/Personnel-Mgr) | "This role holds no access-decision authority — separation of duties." (append "Read-only visibility only." when the role has any `view_*` op) |
| Verdict (ALLOW) | "✓ ALLOW" |
| Verdict (DENY) | "✗ DENY" |
| Override line | "{override name} {detail} (deny override)" |
| Persistent banner | "[DEMO / MOCK]" — non-dismissable, on every screen; simulated/external trust signals carry an inline "[MOCK]" tag |
| Error state | Demo is in-memory and pure-computed — there is no async/network failure surface in Phase 1. If a picker references a missing seed id (developer error), render: "Selection unavailable — this subject/resource is not in the seeded world-state." in the inline destructive style (`bg-destructive/10 text-destructive p-4 rounded-md`, per CLAUDE.md error convention). No toasts. |
| Decision-trace caption (legibility helper) | Plain-prose one-liner under the trace explaining the headline contrast, e.g. "Clearance passes but the COMPUTER tier is too low — per-domain tiers evaluate independently." (DEMO-03 / ENGINE-02) |

Destructive confirmation:

| Action | Confirmation copy |
|--------|-------------------|
| Revoke attribute (Approver) | No modal confirmation. Revocation is event-sourced (`SET_REVOKED`/`REVOKE_ATTRIBUTE`) and reversible by re-granting; the audit log entry IS the record. The amber button color signals caution. |
| Place security hold (Security Officer) | No modal confirmation. Toggle is reversible ("Clear security hold") and event-logged; red button color signals the deny-override severity. |

Rationale: Phase 1 actions are all reversible, event-logged, mock-world mutations. Per CONTEXT D-09 and the proven spike pattern, confirmation is conveyed through button color + the live re-computed trace + the append-only log, not blocking dialogs. (Real destructive confirmations are out of scope — this is a demo island.)
