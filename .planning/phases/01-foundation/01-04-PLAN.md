## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| viewer perception → demo data | A viewer reads decisions/attributes on screen; the risk is mistaking seeded MOCK data (esp. clearance) for a live/real authorization state. |
| operating role → action availability | The UI must only expose actions a role's op set permits; an exposed grant button to a non-Approver would misrepresent the SoD model (ROLE-02). |
| demo island ↔ main app / router | The view must not leak onto the TanStack router or into the main app bundle. |
