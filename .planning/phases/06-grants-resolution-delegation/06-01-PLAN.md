1. isGrantActive — section header: // --- GRANT-01: isGrantActive — time-window check ---
   Signature: export function isGrantActive(grant: PhysicalAccessGrant, now: Date): boolean
   Logic (single boolean expression): (grant.valid_from === null || grant.valid_from <= now) && (grant.valid_until === null || grant.valid_until >= now)
   Both boundaries are inclusive. Null on either side means that boundary is unbounded.

2. resolveGrant — section header: // --- GRANT-04: resolveGrant — ancestor walk, most-specific active grant ---
   Signature: export function resolveGrant(personId: string, zone: ZoneNode, allZones: ZoneNode[], allGrants: PhysicalAccessGrant[], now: Date): PhysicalAccessGrant | null
   Walk logic:
     - Build searchZones: if zone.requires_explicit_auth is true, searchZones = [zone] only (skip ancestor walk entirely). Otherwise searchZones = [zone, ...getAncestors(zone.id, allZones)]
     - Iterate searchZones. For each searchZone: if searchZone.id !== zone.id AND searchZone.zone_type !== zone.zone_type, skip (continue) — zone_type filter applies to ancestors only, never to the target zone itself.
     - For matching searchZone: call allGrants.find(g => g.person_id === personId && g.zone_id === searchZone.id && isGrantActive(g, now))
     - Return first found grant. Return null if none found after full walk.

3. resolveZoneAccess — section header: // --- ACCESS-05: resolveZoneAccess — two-gate entry point ---
   Signature: export function resolveZoneAccess(personId: string, zone: ZoneNode, clearance: Clearance, hasValidEscort: boolean, allZones: ZoneNode[], allGrants: PhysicalAccessGrant[], now: Date): ZoneAccessResult
   Gate 1: const grant = resolveGrant(personId, zone, allZones, allGrants, now). If grant === null, early return { allow: false, gate: "GRANT_LOOKUP", reason: "NO_GRANT" }.
   Gate 2: dispatch by zone.zone_type:
     - "CONTROLLED": return evaluateControlledAccess(true)
     - "RESTRICTED": return evaluateRestrictedAccess(true, clearance, hasValidEscort)
     - default (SECURED): return evaluateSecuredAccess(true, clearance, hasValidEscort)
   NEVER pass hasGrant=false to any Phase 5 evaluate function from inside this function — that would produce gate:"ZONE_TYPE_RULE" on a no-grant path, which is wrong.
   NOTE: For SECURED zones, hasValidEscort maps to the isEscorted parameter of evaluateSecuredAccess — this is annotation-only per D-03; escort does not substitute for clearance in SECURED zones.

4. isDelegateActive — section header: // --- DELEG-01: isDelegateActive — delegate time-window check ---
   Signature: export function isDelegateActive(delegate: ZoneAccessDelegate, now: Date): boolean
   Logic: identical null-boundary pattern to isGrantActive but operating on ZoneAccessDelegate fields: (delegate.valid_from === null || delegate.valid_from <= now) && (delegate.valid_until === null || delegate.valid_until >= now)

Section header style per Phase 5 pattern: // --- LABEL: name — description ---
Function style: named export function, explicit parameter types, explicit return type, no internal Date.now() or new Date() calls.
