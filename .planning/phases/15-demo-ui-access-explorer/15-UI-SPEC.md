## Scope Note — Demo Module Has Its Own Locked Design System

This phase builds entirely inside `frontend/src/demo/`, a **router-isolated demo island** (see
`DemoRoot.tsx`'s own comment) that deliberately does **not** consume the main app's shadcn/Radix
component library. Since Phase 1 (v2.0) the demo has used a small hand-rolled presentational kit
(`frontend/src/demo/components/ui.tsx`: `Card`, `Field`, `Pill`, `Select`, `MockTag`, `DecisionTrace`)
built on raw Tailwind utility classes and a fixed slate/green/red/blue/amber palette. Every phase
since (v2.1 zones, v2.2 digital resources) has extended this same kit rather than introducing
shadcn primitives into the demo tree.

**This phase follows that established convention exactly — no shadcn component (`Button`, `Dialog`,
`Select` from Radix, etc.) is introduced into `frontend/src/demo/`.** All new UI is plain Tailwind +
the existing `ui.tsx` exports, matching `resource-access-explorer.tsx` / `resource-browser.tsx` /
`digital-resources-panel.tsx` pixel-for-pixel in spacing, type scale, and color usage.

## Spacing Scale

Tailwind's default 4px-based scale, as already used throughout `resource-access-explorer.tsx` /
`resource-browser.tsx` / `digital-resources-panel.tsx`. Reuse these exact utility values — do not
introduce new spacing tokens.

| Token | Tailwind utility | Value | Usage (observed in existing demo code) |
|-------|-------------------|-------|-------|
| 2xs | `py-0.5` | 2px | `Pill` vertical padding |
| xs | `gap-2` / `px-2` | 8px | Icon/label gaps, `Pill` horizontal padding, gate-row icon gap |
| sm-exception | `py-1.5` | 6px | Button vertical padding (`Issue grant` button, view-toggle buttons) — **existing exception to the 4/8 scale, reuse verbatim, do not "fix"** |
| sm | `p-3` | 12px | Zone-advisory / error banners, gate-trace compact padding |
| md | `p-4` / `gap-4` | 16px | `Card` padding, top-level `space-y-4` / `grid gap-4` layout rhythm |
| lg | `mt-2` / `space-y-3` | 8–12px | Field-to-field spacing inside a `Card` |

Exceptions: `py-1.5` (6px) on all buttons is an existing project-wide exception — reuse it exactly,
do not round to 8px. No new exceptions introduced by this phase.

## Typography

Exactly as established by `resource-access-explorer.tsx` / `ui.tsx`. Reuse verbatim — no new sizes
or weights.

| Role | Tailwind class | Size | Weight | Line Height |
|------|-----------------|------|--------|-------------|
| Label / meta | `text-xs` (+ `uppercase tracking-wide text-slate-400` for `Card`/`Field` titles) | 12px | 600 (`font-semibold`) | 1.4 (Tailwind default for `text-xs`) |
| Body | `text-sm` | 14px | 400 (regular) / 500 (`font-medium` on gate-row labels) | 1.5 |
| Verdict headline | `text-lg` | 18px | 700 (`font-bold`, DecisionTrace) or 600 (`font-semibold`, ResourceResolutionTrace — **use `font-semibold` for the dataset explorer**, matching `ResourceResolutionTrace`, not `DecisionTrace`'s `font-bold`) | 1.4 |

Only 3 sizes (12/14/18) and effectively 2 weights (400/500 for body content, 600 for
headings/labels/verdict) are in play — matches the 3–4 size / 2 weight ceiling. `font-bold` (700)
exists elsewhere (`DecisionTrace`) but is **not** used by the resource/dataset-tier trace components
— stay consistent with `ResourceResolutionTrace`'s `font-semibold` verdict, per DATA-UI-02's explicit
"styled consistently with `ResourceResolutionTrace`" requirement.

## Color

Dual convention already established in the demo module — reuse both exactly as-is:

| Role | Source | Usage |
|------|--------|-------|
| Dominant (60%) | `bg-slate-50` (page) / `bg-white` (cards) | Page background (`DemoRoot`'s outer div), `Card` surfaces |
| Secondary (30%) | `border-slate-200` / `bg-slate-100` | Card borders, hover/selected row backgrounds (`hover:bg-slate-100`, selected `bg-slate-200`) |
| Accent (10%) | `bg-slate-800 text-white` | **Reserved for:** the primary action button only (`Issue grant`/`Issue dataset grant` submit button) and the active-tab/active-view toggle button background. Nothing else uses this dark-slate accent. |
| Verdict green | `border-green-200 bg-green-50` / `text-green-600` | ALLOW verdict card + passing gate checkmarks only |
| Verdict red | `border-red-200 bg-red-50` / `text-red-600` | DENY verdict card + failing gate checkmarks only |
| Advisory amber | `bg-amber-100 text-amber-900` (`Pill tone="amber"`, `MockTag`) | Non-blocking advisories and `[MOCK]` tags only — never used for ALLOW/DENY |
| Destructive (shadcn token) | `bg-destructive/10 text-destructive` | Error banners only: mutation/API failures, missing-token/unauthorized/error loader states, and the new `ErrorBoundary` fallback (D-14) |

Accent reserved for: the dark-slate (`bg-slate-800`) treatment is used **only** on (1) the
"Issue [dataset] grant" submit button and (2) the active nav-tab / active-view-toggle button. It
must not spread to any other button, badge, or row in this phase's new components.

Second semantic color: `destructive` (shadcn CSS-variable token, `bg-destructive/10
text-destructive`) — reserved for error/failure banners only (API errors, ErrorBoundary fallback,
non-admin/denied issuing feedback), never for the DENY verdict itself (DENY uses the raw
`red-50`/`red-600` pair, a deliberately distinct treatment from the shadcn destructive token — do
not conflate the two).

### Application-only picker (D-01/D-02)

Flat row list, **not** a `Select` dropdown, **not** the collapsible tree:
- Same row shell as `ResourceTreeNodeRow`: `Pill` (tone `"green"`, matching existing `TIER_TONE.APPLICATION`) + name, `flex items-center gap-2 rounded px-2 py-1 text-left text-sm`, selected = `bg-slate-200`, hover = `hover:bg-slate-100`.
- **No chevron/expand affordance** — there is only one tier to pick from (D-02 explicit).
- Source list: `world.digitalResources.applications` (fetched independently per D-04 — do not assume the Digital Resources tab has already populated it).

### Datasets list (right side of the D-06 grid)

- For the selected Application, call `datasetsForApplication(world.datasets.nodes, applicationId)`.
- Each row: `Pill` showing the **literal `dataset_type` string** (`MAILBOX` / `ARCHIVE_ROLE` /
  `DOCUMENT_SITE`, uppercase, no title-casing — matches the existing `TIER_TONE` Pill convention of
  displaying the raw type literal) + dataset `name`. New module-local tone map (per-file copy
  convention, do not share):

  ```
  DATASET_TYPE_TONE: MAILBOX -> "blue", ARCHIVE_ROLE -> "amber", DOCUMENT_SITE -> "slate"
  ```

  (Deliberately avoids `"green"` — reserved for the Application tier's own `TIER_TONE` one level up,
  and avoids implying ALLOW; avoids `"red"` entirely, reserved for DENY.)
- A dataset spanning 2+ `application_ids` appears once per Application it's linked to — no dedup
  (SPEC R1 acceptance).
- Zero datasets → empty-state copy above, not a blank panel.
- Row click selects the dataset, which pre-fills both the explorer and reverse-lookup below (D-05
  single stacked view, no inner sub-tabs).

### Dataset Access Resolution Explorer (`DatasetResolutionTrace`)

Must visually match `ResourceResolutionTrace` exactly: `rounded-lg border p-4`, green/red verdict
background, `text-lg font-semibold` verdict line (`✓ ALLOW` / `✗ DENY`), then an unconditional
`<ul className="mt-3 space-y-1.5">` of all 4 gates in order — **no short-circuit, no hiding
passing/failing gates** (`resolveDatasetAccess` already computes all 4 unconditionally per
13-CONTEXT D-03/D-04).

New `GATE_LABEL` map for the dataset domain (module-local copy in `dataset-access-explorer.tsx`,
per the existing "per-file copy, no shared map" convention):

```
CLEARANCE:    "Clearance"
APP_GRANT_OR: "Application grant"
DATASET_GRANT: "Dataset grant"
VISIBILITY:   "Visibility"
```

**Gap the SPEC leaves open, resolved here:** `resolveDatasetAt` requires an explicit `requiredLevel:
string` parameter — the explorer cannot omit it. Add a **"Required level" field** (`Field` + `Select`,
same shape as the Person/Resource fields), populated dynamically from the selected dataset's
`dataset_type` vocabulary:

- `MAILBOX` → `READ` / `SEND_AS` / `FULL_ACCESS`
- `DOCUMENT_SITE` → `READ` / `CONTRIBUTE` / `FULL_CONTROL`
- `ARCHIVE_ROLE` → `READER` / `CASE_HANDLER` / `ADMIN`

Default to the vocabulary's first (lowest) entry when the dataset selection changes. This field is
required input for the trace, alongside Person + Datetime (`datetime-local`, same pattern as
`ResourceAccessExplorer`'s evaluation-timestamp card).

Reuse `CLEARANCE_TONE` (imported from `access-resolution-explorer.tsx`) for rendering the person's
clearance pill and the dataset's effective classification pill, exactly as
`resource-access-explorer.tsx` does.

### Reverse-lookup view

- One row per person (not per raw grant record) — dedupe by iterating `world.subjects`, calling
  `resolveDatasetAt(...)` per person for the selected dataset at `now`, and including only
  `allow === true`.
- Effective level per included person: `effectiveArchiveCoverage`/`effectiveRankedLevel` result
  (SPEC R3) — display as a `Pill` (reuse `DATASET_TYPE_TONE`-adjacent styling is not needed here;
  use a neutral `Pill tone="slate"` for the level value, consistent with `resource-browser.tsx`'s
  plain-text list rows for grants/delegates).
- Row shape: `personName` + level `Pill`, one `<li>` per person, same `text-sm` list styling as
  `resource-browser.tsx`'s "Active grants" / "Delegates" cards.
- Zero matches → empty-state copy above.
- List order: `world.subjects` array order (no new sort), matching SPEC's "no new sort" ordering rule.

### Admin-gated issuing form (`IssueDatasetGrantSection`)

Mirror `IssueGrantSection` field-for-field, docked directly under the dataset explorer (D-08),
collapsed-by-default `+ Issue new grant` toggle (D-09):

- `Field label="Person"` → `Select` of `world.subjects`
- `Field label="Level"` → `Select` populated from the selected dataset's vocabulary (same list as
  the explorer's Required-level field)
- `Field label="Valid from"` → `input type="date"`, default `todayStr()`
- `Field label="Valid until (optional)"` → `input type="date"`, default empty + helper text "Leave
  blank for permanent grant."
- No delegate-picker field (constraint, SPEC R4 — delegate authority resolves automatically inside
  `canIssueDatasetGrant`)
- Submit button: `bg-slate-800 text-white`, `disabled` + `"Issuing…"` label while pending (D-10 hook
  parity), reset fields on both cancel and success (existing convention)
- Gated on `getStoredUserRole() === "admin"` — non-admin sees only the block message (Copywriting
  Contract above), never the toggle button at all (matches `IssueGrantSection`'s `if (!isAdmin)
  return <p>...</p>` early-return pattern exactly)

### ErrorBoundary (`ui.tsx` addition, D-12/D-13/D-14/D-15)

- Class component; `resetKey` prop = selected dataset's `id` so React remounts fresh on dataset
  change (D-15) — wrap it as `<ErrorBoundary key={selectedDataset.id}>...</ErrorBoundary>` in
  `DatasetsPanel`, wrapping **only** the explorer + reverse-lookup subtree (D-12), never the
  Application/Datasets picker above it.
- Fallback UI: `rounded bg-destructive/10 p-3 text-sm text-destructive` container (same shell as
  every other error banner in the demo), heading text bold, body text regular, plus a `Try again`
  button (same `underline text-xs` treatment as `digital-resources-panel.tsx`'s `Retry` button) that
  clears the boundary's caught-error state.
- Exact copy specified in Copywriting Contract above.
