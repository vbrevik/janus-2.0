<objective>
Extend the store and add the two pieces of genuinely new logic this phase requires: the
`ISSUE_DATASET_GRANT` action/reducer gains `validFrom`/`validUntil` fields (currently
hardcoded to `null`), a new `ErrorBoundary` class component is added to the demo's
hand-rolled UI kit, and a new `useIssueDatasetGrant` hook wraps the (silent-on-denial)
reducer dispatch with `isPending`/`isError` API-mutation-shape parity — using the
dual effect+timeout-fallback pattern that correctly survives React's `useReducer`
bailout on denial (Pitfall 1).

Purpose: This is the foundation Wave 2's `dataset-access-explorer.tsx` builds directly
on — the issuing form there wraps its dispatch in this hook for API-shape parity with
v2.2's mutation hooks rather than calling `dispatch(...)` inline (per D-10), detecting
denial via the audit-log-length comparison (per D-11) since it cannot be built until
`useIssueDatasetGrant` exists and correctly resolves on both the allow and deny paths.
The `ErrorBoundary` — a new generic reusable class component (per D-13) with a static
message + reset button fallback (per D-14) — must exist before Wave 3's `DatasetsPanel`
can wrap the explorer/reverse-lookup subtree in it.
Output: Extended `world-state.tsx` (backward-compatible), new `ErrorBoundary` export in
`ui.tsx`, new `hooks/use-datasets.ts` + its test file.
</objective>
