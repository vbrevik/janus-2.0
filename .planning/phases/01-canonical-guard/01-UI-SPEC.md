# Phase 1 ÔÇö UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note ÔÇö Behavioral Refactor, Design System Locked

**This phase introduces NO new visual design.** Phase 1 (Canonical Guard) is a guard-consolidation
refactor: it deletes the legacy auth-only `protected-route.tsx`, keeps the canonical role-aware
`ProtectedRoute.tsx`, and migrates ~15 route files onto it (per GUARD-01/02/03 and CONTEXT.md
decisions D-01..D-05).

There is **no new page, no new layout, and no new component** to design. The design system is
already **established and locked** (see Design System table below) and is **out of scope to change**.

The only user-visible surfaces this phase can affect are the canonical guard's three runtime states.
Those are documented as **observed contracts from the existing code** (not new design proposals) in
the "Guard State Contracts" section below. The executor must preserve these states exactly ÔÇö the
canonical guard component itself requires no changes; only its importers are migrated.

**No new design tokens, color palette, typography, spacing scale, or copywriting are introduced by
this phase.** The token tables below record the existing, locked system verbatim so downstream
agents (checker, executor, auditor) have the source of truth ÔÇö they are documentation, not new
decisions.

## Guard State Contracts (the only user-visible surfaces in this phase)

Observed from `frontend/src/components/ProtectedRoute.tsx` (read directly). These are the existing
contracts the executor MUST preserve ÔÇö the canonical component is not changed, only adopted.

### State 1 ÔÇö Loading / auth check (in progress)

When `useAuth().isLoading === true`, the guard renders a centered spinner and **nothing else**:

```tsx
<div className="min-h-screen flex items-center justify-center">
  <div className="animate-spin rounded-full h-8 w-8 border-b-2 border-primary"></div>
</div>
```

- Full-viewport (`min-h-screen`), centered, 32px spinner, primary-color stroke (`border-primary`).
- **No "Loading..." text** (the legacy guard had one; the canonical guard does not ÔÇö this is the
  established canonical contract).
- Protected children are NOT rendered during this state.

### State 2 ÔÇö Unauthorized / wrong-role redirect

Behavioral, not a screen. The guard renders no dedicated "access denied" UI; it redirects:

- **Unauthenticated** (`!isAuthenticated`): declarative `<Navigate to="/login" search={{ redirect: <current pathname> }} />`.
  The return URL is preserved so login can send the user back.
- **Authenticated but wrong role** (`!allowedRoles.includes(user.role)`): declarative
  `<Navigate to={redirectTo ?? getDefaultRoute(user.role)} />`. Per `getDefaultRoute`
  (`auth-context.tsx`): `admin Ôćĺ /admin/dashboard`, `enduser Ôćĺ /enduser/tasks`,
  `official Ôćĺ /official/dashboard`, unknown Ôćĺ `/login`.
- There is intentionally **no flash of an "unauthorized" page** ÔÇö the redirect happens on first
  render via `<Navigate>` (synchronous), not a `useEffect`.

### State 3 ÔÇö No-flash-of-protected-content invariant

The guard MUST NOT render protected children before the auth/role check resolves. The canonical
guard satisfies this by returning the spinner during loading and a `<Navigate>` during
unauthenticated/wrong-role checks ÔÇö `return <>{children}</>` is only reached after all checks pass.

This invariant is the core security property of the phase (CLAUDE.md: "do not regress the role-aware
UI guards"). Verification is per CONTEXT.md D-05: manual per-role click-through (log in as admin /
enduser / official, confirm an admin route still redirects a non-admin) ÔÇö no protected content
should ever be visible to a wrong-role user.
