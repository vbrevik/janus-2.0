## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TS resolver ↔ Rust resolver | Drift between the two engines silently diverges client and server access decisions |
| gate evaluation → allow verdict | A mis-mapped gate kind or wrong boundary comparator flips an access decision |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04 | Elevation of Privilege | Unknown gate kind silently ALLOWs | mitigate | GateDescriptor::Unknown + resolver default arm return pass:false UNKNOWN_GATE_KIND (test-asserted) |
| T-11-05 | Elevation of Privilege | Cross-tier inheritance via OWN_TIER_GRANT ancestor walk | mitigate | OWN_TIER_GRANT is a flat resource_id== match; no getAncestors; parity fixture covers it (prohibition) |
| T-11-06 | Tampering | Inclusive boundary off-by-one (`>` vs `>=`) flips boundary access | mitigate | is_window_active uses `valid_until >= now`; golden fixture pins 2026-02-28T23:59:59Z |
| T-11-07 | Spoofing | Non-deterministic `now()` makes parity non-reproducible / time-of-day dependent | mitigate | Resolver takes explicit `now`; grep asserts no Utc::now() inside resolver |
| T-11-SC | Tampering | npm/pip/cargo installs | accept | No new packages (chrono/serde/serde_json already in Cargo.toml; vitest already in frontend) |
