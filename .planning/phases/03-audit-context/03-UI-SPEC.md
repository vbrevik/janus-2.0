## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, `gap-1`, `px-1`, inline pill padding (`px-2 py-0.5`) |
| sm | 8px | Compact element spacing, `gap-2`, `p-2` select padding |
| md | 16px | Default element spacing, `px-4 py-4` card body, `gap-4` grid columns |
| lg | 24px | Section padding, `space-y-6` between audit/context major sections |
| xl | 32px | Layout gaps — not used in demo island (max-width container handles this) |
| 2xl | 48px | Major section breaks — not used in Phase 3 (views are compact) |
| 3xl | 64px | Page-level spacing — not used in Phase 3 |

Exceptions:
- Range slider (`<input type="range">`) fills full width (`w-full`) — no token applies.
- Tab button bar: `px-3 py-2` (= 12px / 8px) — matches 4px grid; throwaway nav (D3-09).
- Touch target minimum: 44px height not enforced (desktop-only demo, no mobile requirement).

## Interaction Contracts

### 4-Tab Navigation (DemoRoot)

- Tab bar: horizontal row of 4 buttons, `flex gap-2`, `mx-auto max-w-5xl px-6 py-2`
- Active tab: `bg-slate-800 text-white rounded px-3 py-2 text-sm`
- Inactive tab: `border border-slate-300 text-slate-600 hover:bg-slate-50 rounded px-3 py-2 text-sm`
- Tab labels: "Decision Explorer", "Federation Hub", "Audit", "Context"
- State: `useState<'decisions' | 'federation' | 'audit' | 'context'>` defaulting to `'decisions'`
- No animation. Instant swap. This is a throwaway nav (D3-09).

### Audit View — Range Slider (AUDIT-01, AUDIT-02)

Primary focal point: the range slider in the Timeline card — it is the interaction entry point from which all other panels derive their state.

- Element: `<input type="range" min={0} max={events.length} value={asOf} />`  
- Default value: `events.length` (= current state / "now") — D3-03
- Label above slider: `Timeline — as of T = {asOf}` (Card title)
- "Current state" chip: visible when `asOf === events.length` — renders `<Pill tone="blue">Current state</Pill>` inline next to the slider label. Hidden otherwise.
- Events applied: `seq ≤ asOf` → `text-slate-700` + "✓ applied" suffix
- Events future: `seq > asOf` → `text-slate-300` + "· future" suffix
- Recomputation: slider `onChange` triggers synchronous recalculation — no loading state needed
- Empty event list: render `<p className="text-sm text-slate-400">No events recorded.</p>` inside the timeline Card

### Audit View — Reconstructed State Panel

- Two-column grid (`grid-cols-2 gap-4`): left = reconstructed subject attributes, right = who-can-access list
- Subject attributes displayed as `Pill` components: clearance (`tone="blue"`), compartments (`tone="slate"`), security hold (`tone="red"` with text "SECURITY HOLD")
- Authorization status row: when authorization rule is wired (D3-04), add `Pill tone={authStatus === 'AUTHORIZED' ? 'green' : 'red'}` with text `AUTHORIZED` or `NOT AUTHORIZED`
- Who-can-access empty state: `<p className="text-sm text-slate-400">Nobody can access this resource at this point in time.</p>`
- Who-can-access populated: `<ul>` of `<li className="text-green-700">✓ {subject.name}</li>`

### Audit View — "Auditor role required" notice (D3-11)

- Render a small inline note inside AuditView (not a modal, not a banner):
  `<p className="text-xs text-slate-400">In production, Auditor role required.</p>`
- Placement: below the event log card, above the reconstructed state panels

### Context View — Policy Divergence Panel (CTX-01, D3-06)

- Layout: two picker cards (subject, resource) in `grid-cols-2 gap-4`; below that, 6-unit policy grid in `grid-cols-3 gap-4` (two rows of 3 units)
- Each unit column: unit name in `text-sm font-semibold`; policy label in `text-xs text-slate-400`; `DecisionTrace` component below
- Divergence is self-evident when some columns show ALLOW (green) and others DENY (red) — no additional highlighting needed
- Picker fields use `Field` + `Select` components matching Spike008 pattern

### Context View — Deployment Toggle Panel (CTX-02, D3-07)

- Layout: two-column `grid-cols-2 gap-4`: left = pickers + toggle, right = `ContextTrace` result
- `ContextTrace` is a local component inside ContextView (matching Spike009 inline pattern) — NOT exported to demo ui.tsx
- Deployment toggle: two buttons, `flex gap-2`, matching Spike009 exactly:
  - Active: `border-blue-300 bg-blue-100 text-blue-800 rounded border px-3 py-1 text-xs`
  - Inactive: `border-slate-300 text-slate-500 rounded border px-3 py-1 text-xs`
- State: local `useState<Deployment>('HOME')` — does NOT dispatch to world-state (D3-07)
- Section heading: `<h2 className="text-sm font-semibold">A · Support obligation (deployment-driven grant)</h2>`

### Context View — Directional Shielding Panel (CTX-03, D3-08)

- Layout: same two-column grid pattern as deployment panel
- Allowlist display: `<Pill tone="red">shielded</Pill>` + `allowlist:` text + `<Pill>` per allowlisted unit
- Section heading: `<h2 className="text-sm font-semibold">B · Directional shielding (deny override on protected data)</h2>`
- Source: shielded resources from `worldState.resources` where `shielded === true` — NOT from local hardcoded list
