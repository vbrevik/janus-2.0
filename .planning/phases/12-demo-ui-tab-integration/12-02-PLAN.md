<objective>
Build the pure data-layer foundations every later Phase 12 plan depends on: a hand-written mapper that denormalizes the Phase 11 backend's flat `/world` response into the frontend's per-node `DigitalResourceWorld` shape, three new `world-state.tsx` reducer actions (`SET_DIGITAL_RESOURCES`, `UPSERT_RESOURCE_GRANT`, `UPSERT_RESOURCE_DELEGATE`), a `QueryClientProvider` in the demo's Vite entry (none exists today), and a one-line `export` fix so the existing classification-color map is reusable by the new components.

Purpose: the backend's `GET /world` response is flatter than the frontend's `NetworkNode`/`PlatformNode`/`ApplicationNode` types (each of which embeds its own `org_links`/`policy_assignments`) — a naive rename-only mapper would silently produce nodes with empty org-links/policy arrays. The reducer has no upsert-by-id action today (only scalar-replace and Set-toggle cases exist), which the issuing forms' duplicate-submit-silence contract requires. No `QueryClientProvider` exists anywhere under `src/demo/` today, so any `useQuery`/`useMutation` call would crash at runtime with "No QueryClient set."
Output: `digital-resource-mapper.ts` (+ test), 3 new reducer actions (+ tests appended to the existing `world-state.test.tsx`), `QueryClientProvider` wired into `demo/main.tsx`, `CLEARANCE_TONE` exported from `access-resolution-explorer.tsx`.
</objective>
