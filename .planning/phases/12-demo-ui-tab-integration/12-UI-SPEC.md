# Phase 11 — UI Design Contract

> Visual and interaction contract for Phase 11: Demo UI & Tab Integration.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> This phase adds a seventh top-level "Digital Resources" tab to DemoRoot,
> with an internal sub-nav following the same button-row pattern as Phase 8
> (Physical Access). Two sub-views: Resource Browser (hierarchy tree + detail
> panel) and Access Resolution Explorer (person + resource + timestamp picker,
> gate-chain trace with amber advisory zone row).
>
> No TanStack route file is touched. `git diff frontend/src/routeTree.gen.ts`
> must be empty after all Phase 11 commits.

## Spacing Scale

Identical to Phase 8 contract — no new exceptions.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-1 py-0.5`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`px-6 py-6`) |
| xl | 32px | Layout gaps (`max-w-5xl` container) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in demo island) |

Exceptions:
- Resource tree indent per depth level: 16px (1rem) per level, using `style={{ marginLeft: depth * 16 }}` — matches Zone Browser pattern exactly
- Sub-nav button row: `px-4 py-2` (16px / 8px) — matches Phase 8 sub-nav

## Typography

Same palette as Phase 8. No new sizes or weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 | Prose explanations, field values, list rows, gate trace detail text |
| Label / badge | 12px (`text-xs`) | 600 (semibold) | 1 | Card section headers (uppercase tracking-wide), Pill badges, Field labels, NSM annotation labels |
| Sub-heading | 14px (`text-sm`) | 600 (semibold) | 1.25 | Resource tree node names, grant list item labels, policy version label |

Two weights only: **400 (normal)** for body prose; **600 (semibold)** for labels, badges, tree node names, verdict. No other weights.

## Interaction Contracts

### ActiveView type extension (DemoRoot)

Add `"digital-resources"` to the `ActiveView` union type in `DemoRoot.tsx`. Add a seventh button to the outer tab bar. Render `<DigitalResourcesPanel />` when `activeView === "digital-resources"`.

### Resource Browser

1. Tree is rendered with Networks as root nodes, expanded to show Platforms (depth 1) and Applications (depth 2). Root nodes start expanded; Applications start collapsed (toggle on Platform node reveals them).
2. Clicking a node with children toggles expand/collapse state (local `useState<Set<string>>`).
3. Clicking any node (leaf or parent) selects it and populates the right-side detail panel.
4. Selected node highlighted: `bg-slate-200`. Unselected hover: `hover:bg-slate-100`.
5. Detail panel sections:
   - Classification Pill: use `CLEARANCE_TONE` mapping from `access-resolution-explorer.tsx`; Applications suffix with " (inherited)" in `text-slate-400 text-xs`.
   - Org links: group by role. Each group heading uses `text-xs font-semibold uppercase tracking-wide text-slate-400`. Each link shows org name + active/expired Pill.
   - Active policy: call `selectActivePolicy(resource.policy_assignments, now)` — show policy label and validity window. If no active policy: `text-sm text-slate-400 "No active policy at this time."`.
   - Active grants: call `activeGrantsForResource(world.digitalResources, resourceId, now)` — list person names. Empty: "No active grants."
   - Delegates: filter `world.digitalResources.delegates` by `resource_id` and `isDelegateActive(d, now)` — list delegate names. Empty: "No delegates."
   - NSM annotations card (Platform only): always render; contains two static Pill badges plus disclaimer and authorizing-authority lookup from the SECURITY_APPROVAL org-link.

### Access Resolution Explorer (Resource)

1. Person dropdown defaults to first subject on mount. Resource dropdown defaults to first resource on mount. Timestamp defaults to current datetime on mount (captured once with `useMemo(() => new Date(), [])`).
2. Resource Select uses a flat option list of all resources: `[...networks, ...platforms, ...applications]` with option label format `[TIER] Name` (e.g. "NETWORK: NorNet Restricted").
3. Timestamp input is `<input type="datetime-local">` — value formatted as ISO string truncated to minutes (`toISOString().slice(0, 16)`). On change: parse to `new Date(value)` and pass as `evalTime` to resolution.
4. Resolution: call `resolveResourceAt(world.digitalResources, personId, person.clearance, person.org_id, resourceId, evalTime)` via `useMemo` — recomputes on any selector or timestamp change.
5. Gate-chain trace updates immediately on any selector change. No submit button.
6. Policy version label shown inside the trace banner: derived from `result.policyVersion`.
7. Zone advisory row rendered when `result.zoneAdvisory !== null`. Uses amber styling — never green/red. "Advisory (non-blocking)" Pill is mandatory.
8. Grant toggle list: filters `world.digitalResources.grants` to the selected person and resource. Checkbox dispatches `TOGGLE_RESOURCE_GRANT` action (field: `resourceGrantId`, not `grantId`).
9. Disabled grants rendered with `line-through text-slate-400` and a parenthetical "(disabled)" label.
10. Clearance Pill + MockTag displayed beside person selector. No manual clearance entry.

### Select sentinel rule

The resource `Select` must always have a non-empty initial value. The first resource in the list is used as the default — never an empty `value=""` option. (Empty `value=""` on `SelectItem` crashes the page — see CLAUDE.md gotcha.)
