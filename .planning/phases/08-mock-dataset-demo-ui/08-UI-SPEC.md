## Spacing Scale

Declared values (multiples of 4 — aligned to Tailwind default scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-1 py-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`px-6 py-6`) |
| xl | 32px | Layout gaps (`max-w-5xl` container) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in demo island) |

Exceptions:
- Zone tree indent per depth level: 16px (1rem) per level, using `ml-4`
- Sub-nav button row: `px-4 py-2` (16px / 8px) — matches standard 4-multiple scale

- Indent: `ml-4` per depth level

## Interaction Contracts

### Zone Browser

1. Zone tree is rendered collapsed to root nodes on mount.
2. Clicking a node with children toggles expand/collapse state (local `useState<Set<string>>`).
3. Clicking any node (leaf or parent) selects it and populates the right-side detail panel.
4. Selected node is highlighted with `font-semibold`. Expand/collapse is independent from selection.
5. Detail panel shows: zone name, level, zone_type Pill, admin_org name, asset_owner_org name, ancestors as breadcrumb Pill chain, active grants as list items, delegates as list items.
6. "Active grants" shows grants filtered to `isGrantActive(grant, now)` and not in `disabledGrantIds`.

### Access Resolution Explorer

1. Person and zone dropdowns default to first available option on mount.
2. Escort dropdown includes "None" as the first option (value `"none"`).
3. On any selector change: `resolveZoneAccess(...)` called via `useMemo` — result updates `DecisionTrace` immediately without user action.
4. Escort validity check: if escort is not "None", call `resolveGrant(escortPersonId, selectedZone, allZones, activeGrants, now)` — `hasValidEscort` is `true` only if a grant is found. Pass result to `resolveZoneAccess`.
5. Grant toggle list: shows all grants where `zone_id` is the selected zone or an ancestor of it. Checkbox unchecked = grant in `disabledGrantIds`. Dispatches `TOGGLE_GRANT` action.
6. `activeGrants` passed to resolution functions = all grants filtered to exclude `disabledGrantIds`.
7. Clearance Pill + MockTag displayed beside person selector — no manual clearance entry.

### Zone Entry Log

1. Zone filter dropdown includes "All zones" as first option. Person filter includes "All persons" as first option.
2. Filter state is local `useState`. Filtering is synchronous (no async).
3. Entry log list ordered by `entry_at` descending (most recent first).
4. For each ESCORT row: call `getActiveVisitorPasses(entry.zone_id, allVisitorPasses, now)` to determine if an active pass exists for the visitor. Display pass badge.
5. `exit_at` rendered as "–" if null (ongoing).
