## 📝 Feature Documentation Structure

Each feature should have:

1. **README.md** - Complete feature documentation
   - Overview and purpose
   - Requirements and data model
   - Architecture and API endpoints
   - Implementation status
   - Testing strategy
   - Completion checklist

2. **AGENT-GUIDE.md** - Agent-specific guidance
   - Assigned agent
   - Workflow and phases
   - Completion criteria
   - Quick start guide

3. **TECHNICAL-DEBT.md** (optional) - Known issues
   - Critical issues
   - Improvements needed
   - Resolution plan

---

## 🎯 Adding New Features

When creating a new feature:

1. Create feature folder: `docs/features/{feature-name}/`
2. Create `README.md` with complete documentation
3. Create `AGENT-GUIDE.md` with agent assignment
4. Create `TECHNICAL-DEBT.md` if there are known issues
5. Update this index file
